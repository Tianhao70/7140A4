Proposed "Global Roots Community" Site Map

This page presents the proposed information architecture for the "Global Roots Community" website. The structure is derived from extensive user research (A2 Persona Interviews, User Stories) and client needs analysis (A2 Client Needs Analysis), aiming to provide comprehensive support and resources for international communities settling in Australia. The sitemap aims to be intuitive, user-centered, and facilitate easy navigation to key resources and community features, aligning with A2 Project Goal 2 ("Accessible Information & Resource Hub").

Purpose: To serve as the central hub for user interaction, peer-to-peer support, and experience sharing among international students and newcomers.

Key Features: Discussion forums categorized by topics (Topics), a feed for latest replies, functionality to create new posts, and a section for users to manage their own posts.

User/Client Need Alignment: Addresses A2 Client Value 1 ("Promote Social Cohesion & Belonging") and A2 Project Goal 3 ("Meaningful Connections & Support Network"). Directly supports user stories from personas like Shihao Wu (seeking peer advice on practical tasks), Rajesh Kumar (professional networking), and Isabella Rossi (connecting with cultural community).

Accessibility Considerations: Clear navigation to sub-sections. Content within posts will adhere to readability guidelines (WCAG 3.1.5). User-generated content will have reporting mechanisms for inappropriate content. Forms for creating posts will be fully accessible (WCAG 3.3.2).

Purpose: To provide categorized forums where users can share experiences, ask questions, and find information related to specific aspects of life in Australia.

Key Features: Threaded discussions, search within topics, ability to follow topics, user profiles linked to contributions.

User/Client Need Alignment: Addresses the need for targeted information sharing expressed by all A2 personas (e.g., Shihao on renting/banking, Nguyen on job market/CVs, Isabella on family settlement/partner's employment). Supports A2 Client Focus Area: "Settling In," "Employment," "Connection".

Accessibility Considerations: Clear heading structure for topic threads (WCAG 1.3.1). Sufficient contrast for text and UI elements within topic lists and posts (WCAG 1.4.3, 1.4.11). Consistent layout for scannability (WCAG 3.2.4).

Discussions on housing, banking, transport, etc. Addresses Shihao's and Nguyen's need for practical settlement advice.

Discussions and resources on finding employment, CV writing, interview skills. Addresses needs of Rajesh, Nguyen, and Isabella (for partner).

Discussions about applying to study programs, scholarships, and academic life for prospective and current international students (A2 Target Audience).

Sharing experiences and tips about student life, campus activities, and balancing studies, relevant to Shihao Wu.

Purpose: Allows users to quickly see the most recent activity across all discussion topics. Features: Chronological feed of replies with links to the original post. Addresses user need to stay updated on active discussions. Accessibility: Links will have clear purpose (WCAG 2.4.4).

Purpose: Interface for users to initiate new discussions or share experiences. Features: Rich text editor (TBD), topic selection, anonymous posting option (ethical consideration). Supports A2 User Story of sharing advice. Accessibility: Form will be WCAG compliant (labels, error handling WCAG 3.3.1, 3.3.2).

Purpose: Allows users to view, manage, and edit posts they have created. Features: List of user's posts, edit/delete options. Supports user control and content management. Accessibility: Clear indicators for editable content and actionable items.

Purpose: To provide users with tools and information to customize their experience and ensure the platform is usable by people with diverse abilities.

Key Features: Options for language selection, visual adjustments (contrast, font size), auditory support information, keyboard navigation guides, and a feedback mechanism for accessibility issues.

User/Client Need Alignment: Directly addresses A2 Client Brief requirement ("Accessibility must be prioritised") and A2 Project Goal 4 ("Ethical, Inclusive & Sensitive Design"). Caters to users like Nguyen Van Cuong needing language support and general WCAG 2.2 AA compliance.

Accessibility Considerations: This entire section is dedicated to accessibility. Each sub-page will itself be highly accessible and clearly explain its features.

Options for users to select preferred language for key content, addressing Nguyen's and Isabella's need. WCAG 3.1.2 Language of Parts.

Settings for contrast modes, font size adjustments (WCAG 1.4.4 Resize text), and text spacing options to assist users with visual impairments. Supports A2 Client Need 10 (WCAG Compliance).

Information on and provision of transcripts or captions for any multimedia content (WCAG 1.2.1, 1.2.2, 1.2.3).

Guidance and features ensuring the site is fully navigable using only a keyboard (WCAG 2.1.1 Keyboard).

A dedicated mechanism for users to report accessibility barriers or provide suggestions for improvement.

Purpose: To provide transparency about the "Global Roots Community" initiative, its mission, and the team behind it, fostering trust.

Key Features: Mission statement, team information, contact details, problem reporting, FAQ.

User/Client Need Alignment: Builds trust (A2 Client Need 6: Safe & Trustworthy Space) and provides essential contact/support channels. Important for users like Rajesh Kumar who value transparency.

Accessibility Considerations: Clear, concise language (WCAG 3.1.5). Easily findable contact information.

Statement outlining the purpose and goals of the website.

Information about the individuals or organization behind the site (linking to real-world presence supports trust, a Reflective Design principle).

Details on potential volunteer or employment opportunities.

Provides clear ways for users to get in touch with site administrators.

A dedicated interface for users to report technical issues or content errors, supporting usability.

Answers to common questions about the website and its services, reducing support load and aiding user understanding.

Key Features: New user sign-up, email/password login, forgot password recovery, and third-party login options.

User/Client Need Alignment: Essential for any community platform requiring user contributions and personalization. Offering multiple login methods caters to user preferences for convenience (A2 User Story derived from Isabella's interview about family preferences).

Accessibility Considerations: All forms will be WCAG compliant (labels, error handling, keyboard access). Secure password practices will be encouraged. Clear instructions for all processes (WCAG 3.3.2).
