AI Reflection: My Experience in A2

The GenAI & MT Usage Framework requires a reflection on how AI was used, how it supported learning, its effectiveness, and ethical considerations. Here's my experience during this assessment:

1. How AI Was Used

I integrated GenAI into several aspects of my A2 workflow:

  • Conceptual Understanding: When I felt uncertain about specific code concepts or lecture materials, I used AI (like Gemini/ChatGPT) to get explanations, helping me consolidate foundational knowledge. For particularly complex lecture sections, AI helped break down the content into more digestible parts.
  • Code Generation & Assistance: I used AI to generate specific, personalized code snippets when exploring visual solutions. A concrete example is when working on the Client Needs Analysis page, I asked for a CSS solution to enhance hierarchy:

    My Prompt:

    Please provide me with a value box solution that increases the sense of hierarchy

    AI's Answer (Summary & Code):

    Distinguishing background colors from shadows. This method uses a background color different from the main background... and can add shadows...

    .value-box { background-color: var(--background-alt-color, #f8f8f8); padding: var(--space-4, 1.5rem); margin-bottom: var(--space-4, 1.5rem); border-radius: 8px; box-shadow: 2px 2px 8px rgba(0, 0, 0, 0.1); } .value-box h3 { margin-bottom: var(--space-3, 1rem); /* ... potential other styles ... */ }

    My Evaluation: This provided a useful visual enhancement for the value boxes.

  • Code Review & Debugging: I utilized AI to review sections of my code, asking it to check for potential flaws, redundancy, or suggest improvements. It was also helpful in debugging, often quickly identifying syntax errors or logic issues I might have overlooked.
  • User Persona Simulation: As required in Week 5, I used GenAI to simulate user personas (International Student, Skilled Migrant, etc.) for conducting interviews to gather user needs.
  • Language Refinement: I used MT tools like Google Translate occasionally to help phrase specific technical terms or ensure clarity when translating my thoughts into written English.

2. How AI Supported Learning

For me, AI acted primarily as a supportive learning partner:

  • It provided **on-demand explanations**, which was invaluable for consolidating basic concepts when I encountered difficulties outside of class time.
  • It allowed me to explore complex topics at my own pace, leading to a **faster, easier, and deeper understanding** than relying solely on static materials.
  • Generating code examples (like the CSS) gave me **practical starting points** that I could then analyze, modify, and learn from, rather than just copying.
  • Assistance with code checking and debugging **freed up cognitive load**, allowing me to focus more on the *logic* and *design principles* rather than getting stuck on minor syntax errors.
  • Simulating diverse user personas helped me gather varied insights efficiently, broadening my understanding of potential user needs for the project.
  • Overall, it significantly increased the **efficiency** of both my learning and my development workflow for this assessment.

3. Critical Evaluation of AI's Effectiveness

AI tools were generally very effective, but it's crucial to recognize their limitations:

  • Strengths: AI excelled at providing quick explanations, generating boilerplate or example code (like CSS styling or basic structures), brainstorming ideas, and rapidly identifying syntax errors during debugging. The value box CSS solution, for example, was a useful and effective suggestion. Simulating personas offered valuable, diverse input quickly.
  • Limitations & Challenges:
    • **Generic/Biased Output:** Persona simulations sometimes yielded generic responses or potential stereotypes, requiring careful prompt refinement and critical evaluation of the generated text.
    • **Contextual Understanding:** AI-generated code occasionally needed significant modification to fit seamlessly within my existing CSS structure (e.g., ensuring variable names matched, styles didn't conflict).
    • **Accuracy Isn't Guaranteed:** Explanations or code snippets weren't always perfect or optimal; verification against course materials and independent testing were essential.
    • **Surface-Level Solutions:** Sometimes AI provided overly simplistic answers, necessitating more specific prompts or deeper follow-up questions to get truly useful insights.
  • My Strategies for Mitigation: I consciously treated AI output as a **first draft or suggestion**, never as a final product. I made it a point to:
    • **Verify:** Cross-reference AI explanations with lecture notes, W3Schools, MDN, or other reliable sources.
    • **Test:** Thoroughly test any generated code in the actual project environment.
    • **Refine & Integrate:** Actively rewrite, adapt, and integrate AI suggestions to match the project's specific needs, my understanding, and the required standards (like accessibility or ethical considerations). This involved modifying prompts iteratively and manually adjusting the outputs.
    • **Focus on Principles:** Use AI for specific tasks, but ensure the core analysis (Client Needs, User Stories, Accessibility, Ethics) remained my own critical thinking process.

4. Ethical Considerations

Using AI responsibly involves several ethical considerations that I kept in mind:

  • **Transparency:** Fully acknowledging the use of AI tools and the extent of their contribution, as documented on this page, is fundamental.
  • **Avoiding Over-reliance & Ensuring Learning:** The most significant ethical challenge for me was ensuring AI enhanced, rather than replaced, my learning. I actively pushed myself to understand the 'why' behind AI suggestions and to perform the core critical analysis myself. It's crucial to **avoid letting AI undermine foundational skill development or critical thinking.
  • **Bias Awareness:** When using AI for persona generation, I tried to be aware of potential biases in the AI's training data and critically reviewed the outputs to ensure they felt reasonably authentic and avoided harmful stereotypes, refining prompts as needed.
  • **Accuracy & Integrity:** I recognized the need to verify AI outputs and take responsibility for the final submitted work, ensuring it accurately reflects the project requirements and my own capabilities.
  • **Data Privacy:** I was mindful not to input any sensitive personal information or confidential project details into public AI tools.

In conclusion, while AI offers powerful assistance, leveraging it ethically and effectively requires ongoing critical engagement, verification, and a clear focus on using it as a tool to augment, not replace, human understanding and creativity.
